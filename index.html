Dataset NSL-KDD

Información del Dataset

NSL-KDD es un conjunto de datos mejorado para la detección de intrusiones en redes. Es una versión refinada del dataset KDD Cup 99, que soluciona algunos de sus problemas inherentes.

Características principales:
  • No tiene registros duplicados
  • Número razonable de registros para entrenamiento y prueba
  • Mejor distribución de registros por categoría
  • Incluye 41 características y 1 variable objetivo

Análisis Exploratorio y Visualización

La distribución de clases en el dataset NSL-KDD es más equilibrada que en el dataset original KDD'99, lo que facilita el entrenamiento de modelos de machine learning.

Observación

Aunque la distribución es más equilibrada, todavía existe un desbalance entre las categorías de ataque, especialmente para U2R y R2L que tienen menos ejemplos. Esto debe considerarse al entrenar los modelos.

El análisis de importancia de características nos permite identificar qué atributos del dataset tienen mayor poder predictivo para la clasificación de intrusiones.

Características Numéricas Clave:
  • src_bytes: Número de bytes de origen a destino
  • dst_bytes: Número de bytes de destino a origen
  • count: Número de conexiones al mismo host
  • serror_rate: Porcentaje de conexiones con errores SYN
Características Categóricas Clave:
  • service: Servicio de red utilizado
  • flag: Estado de la conexión
  • protocol_type: Tipo de protocolo utilizado

El preprocesamiento de datos es fundamental para preparar el dataset NSL-KDD para algoritmos de machine learning.

Pasos de Preprocesamiento:
  1. Limpieza de datos y manejo de valores faltantes
  2. Codificación de variables categóricas (One-Hot Encoding)
  3. Normalización de características numéricas
  4. Balanceo de clases (SMOTE)
  5. División en conjuntos de entrenamiento y prueba
Consideración Importante

La normalización de características es especialmente importante en NSL-KDD debido a la amplia variación en las escalas de las diferentes características (por ejemplo, src_bytes puede variar de 0 a más de 1 billón).

El análisis de correlación nos ayuda a identificar relaciones entre las diferentes características del dataset, lo que puede ser útil para reducir la dimensionalidad.

Correlaciones Fuertes Positivas:
  • dst_host_same_src_port_rate y dst_host_srv_diff_host_rate
  • logged_in y num_root
  • serror_rate y dst_host_serror_rate
Correlaciones Fuertes Negativas:
  • service y dst_bytes
  • dst_host_srv_serror_rate y dst_host_same_srv_rate

Interpretación:

La matriz de confusión muestra que el modelo Random Forest tiene un buen rendimiento en la clasificación de la mayoría de las categorías, con algunas dificultades para distinguir entre ciertos tipos de ataques.

Áreas de mejora:
  • Mejorar la detección de ataques U2R
  • Reducir falsos positivos en la categoría Normal
